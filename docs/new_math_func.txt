ActionC64U MATH1 additions for the native C64U fork
====================================================

Portable source of record: idun_action/lib/math1.act
Native binding reference: lib/math1.act
Target: C64 6502 generated program; no UDOS calls or resident service required.
Numeric base: IEEE-754 binary32 REAL, round-to-nearest/ties-to-even core runtime.

Native UDOS status: all 8 public constants are available through the usable
INCLUDE "MATH1" header, and twenty-three low-level calls are available: FAbs, FSqrt,
FSign, FTrunc, FFloor, FCeil, FRound, FFrac, FMod, FHypot, FPow, FExp, FLn, FLog2, FLog10, FSin, FMin, FMax, FClamp,
DegToRad, RadToDeg, PrintR, and PrintRE. FSign is a dependency-free
selected OBJ helper with canonical-NaN and signed-zero behavior. FMin/FMax use
independently selected OBJ helpers with the complete portable NaN and
signed-zero policy. FTrunc is a dependency-free bitwise binary32 helper that
preserves signed zero, infinities, and NaN payloads. FFloor imports only FTrunc,
rounds finite nonintegers toward negative infinity, and preserves exceptional,
zero, and integral inputs bit-for-bit. FCeil imports FFloor (and transitively
FTrunc), rounds finite nonintegers toward positive infinity, and preserves the
same bit patterns. FRound imports FTrunc, rounds nearest with halfway cases away
from zero, and preserves NaN payloads, infinities, signed zero, and large
integral values. FFrac imports FTrunc and subtraction, returns
`value-FTrunc(value)`, and is safe when source and destination alias. FMod
imports division, truncation, multiplication, and subtraction, computes
`value-FTrunc(value/divisor)*divisor`, and is safe when either operand aliases
the destination. FHypot uses a scaled maximum/minimum calculation, is safe
when either operand aliases the destination, and selects its seven direct
dependencies only when referenced. FExp uses the portable degree-8 range-
reduced algorithm and selects its arithmetic closure only when referenced.
FPow preserves both operands, uses the portable logarithm/exponential algorithm,
and selects its truncation, logarithm, exponential, modulus, subtraction,
multiplication, and transitive arithmetic closure only when referenced.
FLn uses portable exponent/significand range reduction plus a six-term odd
series and selects only subtraction, addition, division, and multiplication.
FLog2 and FLog10 are independent alias-safe wrappers that evaluate FLn into
private storage and divide by embedded binary32 ln(2) or ln(10), selecting only
FLn and division when referenced.
FSin uses a private wrap-to-pi dependency, folds to the primary half-pi
interval, and evaluates the portable degree-11 odd polynomial. Both modules
are alias-safe and selected only when FSin is reachable.
FClamp
selects its comparison/minimum/maximum closure only when referenced, returns
canonical NaN for NaN input or inverted bounds, and
otherwise preserves the selected operand bits. Native source lowering remains
intentionally bounded to named REAL operands. Its pass-K clamp form captures
the three initializer destinations, three arguments, result destination, and
printed value independently, but still requires the fixed four-REAL,
three-initializer, one-call, one-print statement skeleton. The same pass
captures every caller and callee storage role in its bounded finite
two-REAL-parameter comparison/select function, including permuted declaration
and parameter names. The
Idun/Alpine fork implements the complete source library: 8 public constants,
43 public routines, and 1 private helper, producing 51 public help features.
Native delivery still requires wider general REAL function parameters, locals,
expressions, calls, and returns before portable bodies can provide the remaining
20 public routines. See docs/idun_feature_parity.md.

The constants and routines below are public after INCLUDE "MATH1".  The
transcendental routines use portable Action implementations or independently
selected target modules over the binary32 add/subtract/multiply/divide,
conversion, absolute-value, and square-root runtime. They do not use the BASIC
ROM floating-point package.

IEEE and domain behavior
------------------------

- NaN propagates through ordinary calculations and invalid domains return NaN.
- FLn(0) returns negative infinity; FLn(negative) returns NaN.
- FExp overflows to positive infinity and underflows toward positive zero.
- FASin/FACos require -1 through 1.
- FACosh requires a value at least 1.
- FATanh accepts -1 through 1; endpoints return signed infinity.
- FPow accepts a negative base only when the exponent is an exact integer.
- FTrunc preserves NaN payloads, infinities, signed zero, and integral values;
  finite nonintegers truncate toward zero.
- FFloor preserves NaN payloads, infinities, signed zero, and integral values;
  finite nonintegers round toward negative infinity.
- FCeil preserves NaN payloads, infinities, signed zero, and integral values;
  finite nonintegers round toward positive infinity.
- FRound preserves NaN payloads, infinities, signed zero, and integral values;
  finite nonintegers round to the nearest integer with halfway cases away from zero.
- FFrac returns `value-FTrunc(value)`, preserving the sign of every nonzero
  finite fractional part and using the ordinary subtraction policy for exceptional values.
- FMod returns `value-FTrunc(value/divisor)*divisor`; NaN, zero divisor, and
  infinite dividend return canonical quiet NaN, while a finite dividend with
  an infinite divisor is returned bit-for-bit.
- FHypot computes a scaled square root of `x*x+y*y`, returns positive zero for
  two zero inputs, and gives infinity precedence when paired with NaN.
- Division-based functions naturally return infinity or NaN at poles according
  to the binary32 runtime.
- Core IEEE arithmetic is correctly rounded.  Transcendental results use
  range reduction plus finite polynomial/series approximations and are not
  promised to be correctly rounded in the last bit.

Porting checklist for the native C64U tree
-----------------------------------------

1. Port lib/math1.act as source, retaining the public signatures above.
2. Supply the same link-selected binary32 primitives for PrintR, PrintRE,
   FAbs, FSqrt, FSign, FTrunc, FFloor, FCeil, FRound, FFrac, FMod, FHypot, FPow, FExp, FLn, FLog2, FLog10, FSin, FMin, FMax, FClamp, add, subtract, multiply, divide, compare, INT
   conversion, and REAL conversion.
3. Keep _MathWrapPi private; help/catalog builders intentionally ignore names
   beginning with an underscore.
4. Add the constants and all public signatures to ACTEDIT/ACTHELP library help.
5. Run known-value target tests for positive and negative FATan, all FATan2
   quadrants, FSin/FCos/FTan, logs, powers, and hyperbolic functions.  The Idun
   fork's VICE regression executes seventeen representative results on a 6502.

Small example
-------------

INCLUDE "MATH1"

MODULE MAIN

PROC MAIN()
    REAL angle,x,y
    angle=DegToRad(45.0)
    x=FCos(angle)
    y=FSin(angle)
    PrintR(FHypot(x,y))
    RETURN
ENDPROC
